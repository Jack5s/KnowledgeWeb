Spring Boot

简介

Spring Boot提供了一个简易的方式用于MVC框架的搭建，是对于Servlet的一个扩展，在其基础上增加了映射的一些参数

官网：https://spring.io/projects/spring-boot

Spting Boot配置

Spring Boot需要进行一系列配置，通常使用Maven进行依赖的添加。关于Maven

Maven配置
 
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
    <dependency>
        <groupId>com.jayway.jsonpath</groupId>
        <artifactId>json-path</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
Application配置

Spring Boot必须有一个启动类，在main/java中新建包，并在包中新建类Application：

@SpringBootApplication
public class Application extends SpringBootServletInitializer {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
Controller配置

与Application同一package（或子package）中新建类：

@RestController
public class TestController {
    @RequestMapping(value = "/checkUser")
    public String checkUser(@RequestParam(name = "user",value = "user",required = false)String userName){
        return  "ok";
    }
}

@RequestMapping中value是网络地址映射，与servlet中@WebServlet类似

@RequestParam表示输入参数
发布到Tomcat

Spring Boot有内置Tomcat服务器，这就意味着器无法与其它的web程序同时发布到Tomcat上，但可以通过一定修改使其发布在Tomcat上：

Maven配置

添加sevlet依赖

<dependency>
    <groupId>org.apache.tomcat</groupId>
    <artifactId>tomcat-servlet-api</artifactId>
    <version>8.0.36</version>
    <scope>provided</scope>
</dependency>
Application更改

Spring Boot通过Application类启动，因此可以通过改写Application类来更改启动设置：

public class Application extends SpringBootServletInitializer {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
